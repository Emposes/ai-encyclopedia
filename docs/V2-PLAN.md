## Architecture: Track × Level

- **Track** = domain (the directory + hub grouping). **Level** = existing badge: INTRO (Foundations) / CORE (Practitioner) / ADVANCED (Frontier).
- The hub groups tracks into 5 **Parts**. Onboarding routes by intent (which tracks) + depth (level); the full index still scours everything.
- Central `content.json` manifest becomes the single source of truth (index, routing, onboarding, search, videos, bibliography). Generated by build from per-page `data-track`/`data-level`/`data-refs`.
- New dirs require one allowed shared edit: extend the progress regex in `shared.js` from `/(chapters|ml|prompting|agents)/` to include the new tracks.
